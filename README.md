# 業務委託ドライバー請求書自動作成システム(プロトタイプ)

業務委託ドライバーがサイト上で稼働実績を入力し、管理側が控除・インセンティブ・経費を割り振ったうえで、
ドライバー名義の適格請求書(インボイス制度対応)を自動発行できる動くプロトタイプです。

ローカルのMac上で試す分にはNode.js標準機能のみで動き、**npm install は不要**です。Node.js(18以上推奨)
さえ入っていればすぐに起動できます。クラウド(Render)に公開してデータを永続化する場合のみ、MongoDBに
接続するための1つの依存パッケージが必要になります(詳しくは後述の「インターネットに公開する」を参照)。

## 起動方法

```
node server.js
```

起動すると以下が表示されます。

```
ドライバー画面: http://localhost:3000/driver/login.html
管理画面      : http://localhost:3000/admin/login.html
```

初回起動時に自動で以下が作成されます。

- 管理者アカウント: `admin@example.com` / `admin1234`(必ずログイン後にパスワードを変更する運用にしてください。本プロトタイプにはパスワード変更画面はないため、変更が必要な場合は `data/db.json` の `admins[0].password_hash` を差し替えるか、開発者に依頼してください)
- 会社情報(請求書の宛先): 「株式会社サンプル運送」という仮の情報。**管理画面の「会社情報設定」から実際の会社名・住所に変更してください。**

データは `data/db.json` にJSON形式で保存されます(簡易データストア)。バックアップはこのファイルをコピーするだけで可能です。

## 業務フロー

### ① ドライバー側 (`/driver/login.html`)

1. 新規登録してログイン(氏名・メール・パスワード)
2. プロフィール画面で住所・銀行口座・インボイス登録番号(適格請求書発行事業者の場合)を入力
3. 「月次入力」タブで対象月を選び、勤務日数・稼働単価(円/日)を入力 → 売上額を自動計算。ガソリン代の単価(円/日)を入力すると勤務日数×単価で自動計算されます。その他稼働だけは出勤日数を別に入力する欄があり、その他稼働の出勤日数×単価で計算されます
4. 控除項目(車両代・事務手数料・車両修繕代・前借り分返済)や経費(消耗品費・車両修繕)を自己申請できる(管理側が確認・調整)
5. インセンティブは閲覧のみ(付与は管理側で行う)
6. 発行された請求書は「請求書一覧」タブから閲覧・印刷可能

### ② 管理側 (`/admin/login.html`)

1. 管理者アカウントでログイン
2. 対象月を選び、ドライバー一覧で売上額・入力状況を確認
3. 各ドライバーの詳細画面で、控除項目・インセンティブ・経費を確認/追加/金額調整して保存
4. 「この月の請求書を発行する」で、ドライバー名義の請求書を自動生成
5. 「請求書一覧」画面から発行済み請求書を確認し、印刷・PDF保存、メール送付、送付済みステータスへの変更が可能

## 請求書について(消費税は加算しない仕様です)

- 本システムは消費税を加算せず、各項目の金額をそのまま合算・差引して請求する仕様です。
  稼働費・ガソリン代・その他稼働・インセンティブ・経費はすべてそのまま合算され、控除項目はそのまま差し引かれます。
- ドライバーがプロフィールで「適格請求書発行事業者に登録している」にチェックし登録番号(T+13桁)を入力すると、
  請求書に登録番号が参考情報として表示されますが、消費税額の表示は行いません。
  **消費税を課税する取引が含まれる場合、この書類は適格請求書の要件(税率ごとの消費税額表示)を満たさない点に
  注意してください。実際の税務処理(課税・非課税・免税事業者の経過措置など)については税理士にご確認ください。**
- 控除項目(車両代・事務手数料・車両修繕代・前借り分返済)は内部精算項目として、合計から差し引く形で表示しています。

## インターネットに公開する(無料でみんなにアクセスさせ、データも消えない構成)

自分のMacで動かしている間しか使えないと不便なので、**Render(無料Webホスティング)+ MongoDB Atlas(無料
データベース)** の組み合わせでクラウドに載せます。この構成なら、Renderのサーバーが再起動・再デプロイされても
ドライバー登録や請求書のデータは消えません(データはMongoDB Atlas側に保存されるため)。

このシステムは `MONGODB_URI` という環境変数が設定されているとMongoDBにデータを保存し、設定されていなければ
今まで通りローカルのJSONファイルに保存します(ローカルで試すだけならこれまで通り `node server.js` だけでOK、
npm installも不要です)。クラウドに公開するときだけ、下記の手順でMongoDB Atlasと接続します。

### 手順

1. **MongoDB Atlasアカウントを作成**(https://www.mongodb.com/cloud/atlas/register )
   - 無料の「M0」クラスターを作成(リージョンはどこでもよい。東京がなければ近いリージョンを選択)
   - 「Database Access」でユーザー名とパスワードを作成(接続文字列で使うので控えておく)
   - 「Network Access」で `0.0.0.0/0`(どこからでも接続可)を追加(Renderのサーバーは接続元IPが固定ではないため)
   - クラスター作成後「Connect」→「Drivers」から接続文字列(`mongodb+srv://...`)をコピー。
     `<password>` の部分を作成したパスワードに置き換え、末尾に `/driver_invoice_system` を追加しておく
     (例: `mongodb+srv://user:pass@cluster0.xxxxx.mongodb.net/driver_invoice_system`)
2. **GitHubアカウントを作成**(https://github.com )→ 新しいリポジトリを作成(Private/Publicどちらでも可)
3. ダウンロードした `driver-invoice-system.zip` を展開し、リポジトリのページで「Add file」→「Upload files」から
   フォルダの中身をまとめてドラッグ&ドロップしてアップロード → 「Commit changes」
4. **Renderアカウントを作成**(https://render.com )→ GitHubアカウントでログインすると連携が簡単
5. Renderのダッシュボードで「New +」→「Web Service」→ 先ほどのGitHubリポジトリを選択
6. 設定画面で以下を入力
   - Runtime: `Node`
   - Build Command: `npm install`(MongoDBに接続するためのパッケージを取得するため、今回は必須です)
   - Start Command: `node server.js`
   - Instance Type: `Free`
   - Environment Variables に以下の2つを追加
     - `SESSION_SECRET`: 適当なランダムな文字列(例: 好きな長い英数字)
     - `MONGODB_URI`: 手順1でコピーしたMongoDB Atlasの接続文字列
7. 「Create Web Service」をクリックしてデプロイ開始 → 数分待つと `https://(サービス名).onrender.com` のような
   URLが発行されます。このURLをドライバー・管理者に共有してください。

補足:
- 起動時のログに `データストア: MongoDB (永続化あり)` と表示されていれば、正しくMongoDBに接続できています。
  もし `Cannot find module 'mongodb'` のようなエラーが出る場合は、Build Commandが `npm install` になっているか確認してください。
- 無料枠は15分アクセスがないとスリープし、次のアクセス時に起動まで30秒〜1分ほどかかります(社内利用なら実用上問題になりにくいです)。
- MongoDB Atlasの無料枠(M0)は512MBまで・期限なしで使えます。ドライバー数十人規模の請求データであれば十分な容量です。
- 手元でテストしたいときは、これまで通り環境変数なしで `node server.js` を実行すればJSONファイルモードで動きます
  (MongoDBの設定は不要です)。

## 請求書をメールで自動送付する(会社のメールアドレス → 銀行のメールアドレス)

管理画面の請求書一覧・ドライバー詳細画面にある「メールで送付」ボタンを押すと、会社のメールアドレスから
銀行(経理)への送付先メールアドレスへ、請求書の内容をメール本文(HTML)として送信します。

### 準備

1. **送付先メールアドレスを設定**: 管理画面の「会社情報設定」で「銀行への送付先メールアドレス」を入力して保存
2. **送信元メールアドレスの準備(Gmailの場合)**:
   - 送信に使うGoogleアカウントで2段階認証を有効にする
   - https://myaccount.google.com/apppasswords で「アプリパスワード」を発行する(16桁の文字列が表示されます)
   - 通常のGoogleアカウントのパスワードではなく、必ずこのアプリパスワードを使います
3. **Renderの環境変数に以下を追加**(Environment画面から追加 → 保存すると再デプロイされます)
   - `SMTP_USER`: 送信に使うGmailアドレス(例: yourcompany@gmail.com)
   - `SMTP_PASS`: 手順2で発行した16桁のアプリパスワード
   - (`SMTP_HOST` / `SMTP_PORT` は省略可。省略時は自動的にGmail用の設定 `smtp.gmail.com:465` が使われます)

設定後、請求書一覧画面の「メールで送付」ボタンを押すと送信されます。送信に成功すると自動的に「送付済み」
ステータスになります。送信に失敗した場合は、画面にエラーメッセージが表示されます(送付先メールアドレス未設定、
SMTP設定不備など)。

## 控除・インセンティブ・経費の項目を変更したい場合

`lib/constants.js` を編集してください(サーバー再起動不要、リクエストのたびに読み込みます)。

```js
const DEDUCTION_CATEGORIES = ['車両代', '事務手数料', '車両修繕代', '前借り分返済'];
const INCENTIVE_CATEGORIES = ['繁忙期インセンティブ', 'SNSインセンティブ', 'リクルートインセンティブ', 'その他'];
const EXPENSE_CATEGORIES = ['消耗品費', '車両修繕'];
```

## このプロトタイプの制約(本番運用前に確認してください)

- データストアは既定でJSONファイル(`data/db.json`)、`MONGODB_URI` 設定時はMongoDBです。同時アクセスが
  非常に多い・データ量が非常に多い場合は、より本格的なデータベース構成への移行を検討してください。
- セッションは署名付きCookieに保存しています。`SESSION_SECRET` 環境変数を設定しないと、サーバー再起動のたびに
  秘密鍵が変わり、ログインし直しが必要になります。本番運用時は固定の `SESSION_SECRET` を設定してください。
- ドライバーの新規登録は誰でも可能な状態です。実運用では管理側が招待制にする、メール確認を入れる等の対応を
  検討してください。
- HTTPS化やレート制限、監査ログなど、本番セキュリティ要件は別途対応が必要です。
- 請求書はPDF専用ライブラリではなく、ブラウザの「印刷 → PDFに保存」機能を使う想定です(依存パッケージを
  増やさないための設計)。

## ディレクトリ構成

```
server.js              起動エントリポイント
lib/miniapp.js          依存なしの超軽量Webフレームワーク
lib/db.js                データストア(JSONファイル / MongoDBのどちらも対応)
lib/calc.js              請求金額の計算ロジック
lib/constants.js         控除・インセンティブ・経費の項目マスタ
lib/invoiceTemplate.js   請求書印刷用HTML・メール本文の生成
lib/mailer.js            請求書メール送信(nodemailer)
lib/passwords.js         パスワードハッシュ(scrypt)
lib/seed.js              初期データ投入
lib/auth.js              認証ミドルウェア
routes/driver.js         ドライバー向けAPI
routes/admin.js          管理者向けAPI
routes/print.js          請求書印刷ページ
public/driver/           ドライバー画面
public/admin/            管理画面
data/db.json             データ本体(初回起動時に自動作成)
```
