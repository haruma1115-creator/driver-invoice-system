# ドライバー希望休・自動シフト作成システム

Renderにデプロイして、ドライバー陣・管理者にそれぞれ別のURLで共有するための構成です。

## 画面のURL(デプロイ後)
- ドライバー用: `https://【あなたのRenderのURL】/driver`
- 管理者用: `https://【あなたのRenderのURL】/admin`
- ルート(`/`)にアクセスすると自動的に `/driver` にリダイレクトされます

管理者用のURLはドライバー陣には共有しないようにしてください(現時点ではパスワード等のログインはなく、URLを知っていれば誰でも開ける簡易な仕組みです)。

## 構成
- `server.js` … 画面配信 + データ保存 + 自動シフト作成のロジック + チャット指示の解析(すべてサーバー側)
- `public/driver.html` … ドライバー用画面(①全体のシフト表 ②希望休送付)
- `public/admin.html` … 管理者用画面(①全体のシフト表 ②希望休の確認 ③ドライバー追加)
- `public/style.css` / `public/common.js` … 共通スタイル・日付ユーティリティ
- `data.json` … 保存データ(初回起動時に自動生成されます)

## シフト表記
- `◯` 終日勤務
- `C1` 半日勤務(午前)
- `C2` 半日勤務(午後)
- `×`(グレー) 休み
- `×`(赤) ドライバーが希望した休み

## ドライバーのログインについて
ドライバー用画面(`/driver`)は、名前+パスワードでのログインが必要です。
- パスワードは管理者画面の「③ドライバー追加」タブ、各ドライバーの🔑ボタンから設定します(未設定の場合はログインできません)
- ログインに成功すると、その端末には次回から自動でログイン状態が保持されます(ログアウトボタンで解除できます)
- パスワードはサーバー側でハッシュ化して保存していますが、簡易的な仕組みのため、金融システム等と同等の強固なセキュリティではない点にご留意ください

## チャットでの指示出しについて(管理者画面)
管理者画面の「①全体のシフト表」タブ下部に、シフトへの指示をチャット形式で入力できる欄があります。
例:「鈴木さんは体調不良が多いから、週3日の勤務にしたい」

- **標準動作(APIキー不要)**: 「週◯日」のような具体的な数値を含む指示は、簡易なキーワード解析でそのドライバーの週あたり勤務日数上限として自動反映されます(次回の自動シフト作成から反映)。数値が読み取れない指示は、そのドライバーへの備考として保存されるのみで、自動反映はされません。
- **より柔軟な解釈をしたい場合**: RenderのEnvironment設定で環境変数 `ANTHROPIC_API_KEY` にAnthropicのAPIキーを設定すると、Claude APIを使ってより自然な日本語の指示から意図を読み取るようになります(この場合、APIキーの利用料が発生します)。

## ローカルでの動作確認
```bash
npm install
npm start
```
`http://localhost:3000/driver` と `http://localhost:3000/admin` を開いて確認できます。

## Renderへのデプロイ手順
1. このフォルダの中身をGitHubリポジトリにpush(またはアップロード)する
2. Renderのダッシュボードで **New +** → **Web Service** を選択
3. 対象のGitHubリポジトリを接続
4. 設定
   - **Build Command**: `npm install`
   - **Start Command**: `npm start`(※`npm install`ではなく`npm start`です)
   - **Environment**: Node
5. 「Create Web Service」でデプロイ開始
6. デプロイ完了後に発行されるURLに `/driver` または `/admin` を付けてアクセス

## データの永続化について(重要)
無料プランのRenderは、再デプロイやスリープ復帰の際に `data.json` の内容がリセットされる場合があります。
運用を安定させたい場合は、Renderの **Persistent Disk**(有料)を追加して環境変数 `DATA_DIR` にマウントパスを設定するか、データベースへの保存に切り替えることをおすすめします。

## 運用の流れ
1. 管理者画面の「③ドライバー追加」でドライバーを登録(希望休の提出状況に関係なく、いつでも追加・削除できます)
2. ドライバーは「②希望休送付」から希望休を提出
3. 管理者は「必要人数」を設定し、必要に応じて「シフトへの指示出し」でドライバーごとの条件を調整
4. 「この月を自動シフト作成」を実行
5. 生成されたシフト表を確認し、必要であればセルをクリックして手動調整
